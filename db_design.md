# 통합 회원가입 및 커뮤니티 데이터베이스(DB) 설계서

이 문서는 영진전문대학교 회원가입 및 커뮤니티 페이지에서 공통으로 사용되는 회원 정보를 저장하기 위한 통합 데이터베이스 설계서입니다. Supabase (PostgreSQL) 데이터베이스를 기반으로 구축되었습니다.

---

## 1. 테이블 정의 (Table Definition)

- **테이블명**: `members`
- **설명**: 통합 회원 정보를 관리하는 테이블

| 순서 | 컬럼명 (Column Name) | 데이터 타입 (Data Type) | PK/FK | Null 여부 | 제약조건 (Constraints) | 설명 (Description) |
| :--- | :--- | :--- | :---: | :---: | :--- | :--- |
| 1 | `number` | `BIGINT` | PK | NN | GENERATED BY DEFAULT AS IDENTITY | 회원 고유 번호 (PK) |
| 2 | `user_id` | `VARCHAR(50)` | - | NN | UNIQUE, 아이디 형식 검증 | 로그인 아이디 (중복 불가능) |
| 3 | `password` | `VARCHAR(255)` | - | NN | 최소 8자 이상 | 암호화된 비밀번호 (단방향 해시) |
| 4 | `name` | `VARCHAR(50)` | - | NN | 실명 입력 | 사용자 실명 (이름) |
| 5 | `nickname` | `VARCHAR(50)` | - | NN | - | 게시판 및 화면 표시용 닉네임 |
| 6 | `email` | `VARCHAR(100)` | - | NN | UNIQUE, 이메일 형식 검증 | 이메일 주소 (비밀번호 찾기 등에 활용) |
| 7 | `phone` | `VARCHAR(20)` | - | YES | 형식: `010-XXXX-XXXX` | 연락처 번호 (선택 사항) |
| 8 | `is_admin` | `BOOLEAN` | - | NN | DEFAULT false | 최고 관리자 권한 여부 |
| 9 | `created_at` | `TIMESTAMPTZ` | - | NN | DEFAULT now() | 최초 회원가입 일시 |
| 10 | `updated_at` | `TIMESTAMPTZ` | - | NN | DEFAULT now() | 회원정보 마지막 수정 일시 |

> **NN**: Not Null (빈 값 허용 안 함)  
> **PK**: Primary Key (기본키)  
> **UNIQUE**: 중복 값 허용 안 함
> **YES**: Nullable (빈 값 허용)

## 2. SQL DDL (Data Definition Language)

테이블을 생성하기 위한 SQL 스크립트입니다. (PostgreSQL / Supabase 기준)

```sql
CREATE TABLE public.members (
    number BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    user_id VARCHAR(50) UNIQUE NOT NULL,
    password VARCHAR(255) NOT NULL,
    name VARCHAR(50) NOT NULL,
    nickname VARCHAR(50) NOT NULL,
    email VARCHAR(100) UNIQUE NOT NULL,
    phone VARCHAR(20),
    is_admin BOOLEAN DEFAULT false NOT NULL,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT TIMEZONE('utc'::text, NOW()) NOT NULL,
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT TIMEZONE('utc'::text, NOW()) NOT NULL
);
```

## 3. 컬럼별 세부 고려 사항 및 보안 설계

### ① `number` (회원 번호)
- 회원 관리를 위한 기본 식별자(Primary Key)입니다.
- 대용량 데이터 처리를 위해 `BIGINT` 형식을 사용하며, Supabase/PostgreSQL의 `GENERATED BY DEFAULT AS IDENTITY` 속성으로 자동 증가시킵니다.

### ② `user_id` (아이디)
- 중복 가입을 방지하기 위해 `UNIQUE` 제약조건을 부여합니다.
- 영문과 숫자만 포함하도록 정규식 검증(Javascript 및 Backend)을 필수로 수행합니다.

### ③ `password` (비밀번호)
- **보안 필수**: 비밀번호는 절대 평문(Plain Text)으로 DB에 저장하면 안 됩니다.
- 반드시 **단방향 해시 알고리즘**(`bcrypt`, `Argon2` 또는 `PBKDF2`)을 적용하여 암호화한 후 저장해야 합니다.
- 프론트엔드의 경우 Mock 해싱을 이용하여 전달하며, 실제 백엔드에서는 강력한 해싱 솔루션을 제공합니다.

### ④ `name` (이름) & `nickname` (닉네임)
- 회원 이름(실명)과 서비스 내에서 활용할 닉네임 정보를 저장합니다.
- 커뮤니티 게시글 및 댓글 작성 시 실명이 아닌 닉네임을 노출하여 개인정보를 보호합니다.

### ⑤ `email` (이메일)
- 비밀번호 분실 시 본인 확인용 및 알림 수신 등에 활용됩니다.
- 입력값에 대한 이메일 형식 검증(Regular Expression)을 반드시 거칩니다.

### ⑥ `phone` (전화번호)
- 대시(`-`)를 포함하여 일관된 형식(`010-XXXX-XXXX`)으로 포맷팅하여 저장합니다.
- 커뮤니티 전용 회원가입 등 전화번호가 제공되지 않는 경로가 있을 수 있으므로 `Nullable`로 설정합니다.

### ⑦ `is_admin` (관리자 권한)
- 회원 계정의 최고 관리자 권한 부여 여부를 결정하는 Boolean 타입 필드입니다.
- 관리자 계정만 공지사항 등록 및 특정 관리 통제를 할 수 있도록 권한 분리 모델을 적용합니다.
